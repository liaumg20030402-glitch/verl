\textbf{病历质控规则奖励 GRPO 训练（端到端打通 post-training 链路）}
\begin{itemize}
    \item \textbf{数据构造与归因分析:}基于线上 bad case 对 Qwen3.5-MoE 思维链进行归因分析,抽象出覆盖推理正确性、规则逻辑遵循、逻辑连贯性、知识正确性、关键步骤覆盖等 10 个维度的 step-level 错误标签;抽样分析 500 条 CoT、15{,}249 个推理步骤,定位到错误主要集中在前 3 步,为奖励函数设计提供依据。
    \item \textbf{规则奖励函数设计:}基于 JSON 字段抽取 + 结论匹配的混合规则实现 \texttt{compute\_score\_blzk\_rule},覆盖格式合法性、关键字段完整性、最终结论一致性等多重约束,产出 0/1 二元奖励信号,并附加可观测的 \texttt{judge\_reason} 分类(ok / mismatch / json\_not\_found / json\_parse\_failed 等)用于训练监控。
    \item \textbf{GRPO 训练实现:}基于 verl 框架在 H200 多机集群上跑通 Qwen3.5-35B-A3B (MoE) 的 GRPO 训练,配置 TP=2 / EP=8 / PP=1 并启用 Megatron 后端 + vLLM 异步 rollout;采用 dapo reward manager 配合 overlong reward shaping 防止长 response 截断带来的奖励噪声。
    \item \textbf{训练效果:}在内部病历质控测试集上,reward mean 从 \textbf{[待填:基线值]} 提升至 \textbf{[待填:训练后值]}(\textbf{[待填:相对提升 \%]}),judge\_reason 中 \texttt{json\_not\_found} 比例从 \textbf{[待填]\%} 下降至 \textbf{[待填]\%},验证了规则奖励 + GRPO 在结构化输出任务上的有效性。
\end{itemize}

\textbf{医考选择题生成式奖励模型 GRPO 训练（进行中）}
\begin{itemize}
    \item \textbf{数据构造:}整理构造 12{,}950 条中文多选题与 65{,}293 条中英文单选题,将单选题筛选并改写为开放式问答,最终保留 53{,}668 条高质量医考开放式 QA(中文 39{,}095 / 英文 14{,}573);组织多模型采样回答并用 LLM 自动打分,构建训练集 20{,}000 query × 8 answers = 160{,}000 条偏好数据候选集与 4{,}000 条验证集。
    \item \textbf{生成式奖励模型(GenRM)设计:}相比规则匹配难以处理开放式答案的语义等价问题,采用 Qwen2.5-7B-Instruct 作为 GenRM 后端,以 \texttt{/v1/chat/completions} 接口异步并发打分,在 verl 中通过 \texttt{REWARD\_MODE=genrm} 同时启用 \texttt{reward\_model.rollout} 与 \texttt{custom\_reward\_function} 双管道。
    \item \textbf{对比实验设计(计划):}规则匹配 baseline / 判别式 RM / 生成式 RM 三组对比,关注最终 acc、reward hacking 倾向、推理步骤合理性等指标,\textbf{[待补充实验结果]}。
\end{itemize}

\textbf{Qwen3.5-MoE 多机 RL 训练环境搭建与工程化}
\begin{itemize}
    \item 在 H200 集群(2 机 16 卡,141GB/卡)部署 Swift / verl / vLLM / Megatron-LM 全套 post-training 工具链,通过自研 ky 调度器手工编排 Ray 集群(RANK=0 head + RANK>0 worker),修复多机部署中 \textbf{多网卡 IP 解析、Ray 节点资源 race、shell 解析 Ray INFO 污染、Hydra struct mode、flashinfer JIT 在 NFS 上并发竞争}等 6 类关键工程问题,打通 35B MoE GRPO 训练。
    \item 针对 Qwen3.5 GDN 线性注意力不支持 packed sequence (THD) 格式的架构限制,锁定 \texttt{use\_remove\_padding=False} + \texttt{use\_dynamic\_bsz=False} 走 BSHD 路径,并通过节点本地编译缓存(TRITON / FLASHINFER / XDG\_CACHE) 规避 NFS race。
    \item 熟悉 RL 训练框架的核心配置与兼容性权衡:三层 batch (train / mini / micro)、KL 加 loss vs 加 reward、低方差 KL 估计器(k3)、PPO clip-higher、动态 batch、reward manager 选型(naive vs dapo)、attention backend 选型等。
\end{itemize}
