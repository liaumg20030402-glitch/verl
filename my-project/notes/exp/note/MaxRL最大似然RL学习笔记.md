为什么除了减去均值，还要“除以标准差”？减去均值很好理解：为了在这一组回答中“比个高低”。高于平均分的给正奖励，低于平均分的给负惩罚。那为什么还要多此一举去除以标准差呢？核心原因是为了稳定训练过程（尺度统一）。在统计学上，这一步被称为 Z-score 标准化（零均值，单位方差）。打个比方：情境 A（简单题）： 模型生成的 4 个回答得分分别是 0.98, 0.99, 1.0, 1.01。这组回答的差异极其微小。情境 B（争议题）： 模型生成的 4 个回答得分分别是 0, 5, 10, 15。这组回答的差异极大。如果不除以标准差：在情境 A 中，最好的回答（1.01）减去均值（0.995）后，优势只有 +0.015，模型只能拿到微弱的奖励梯度。而在情境 B 中，最好的回答（15）减去均值（7.5）后，优势高达 +7.5。这会导致模型在训练时，被情境 B 这种波动巨大的题目疯狂牵着鼻子走，参数更新一脚深一脚浅，极易引发训练震荡（不收敛）。除了标准差之后：不管原来分数的波动是 0.01 还是 100，算出来的优势 $A_i$ 基本都会被强行压缩到 -2 到 +2 之间。这就好比给模型戴上了一个“平滑器”，告诉它：“无论这道题绝对分数的差距有多大，只要你是这组里的第一名，你就拿固定的标准奖励；是最后一名，就拿固定的标准惩罚。” 这种剥离了绝对数值、只看相对排名位置的做法，是大模型强化学习能极其稳定收敛的终极秘诀。
当你把一组数据减去均值、并除以标准差之后（这在统计学叫计算 Z-score），这组新数据的平均值一定会变成 0，标准差一定会变成 1。如果大模型生成的回答得分大致符合正态分布（钟形曲线），那么根据统计学法则：约 68% 的数据会落在 $[-1, 1]$ 之间。约 95% 的数据会落在 $[-2, 2]$ 之间。约 99.7% 的数据会落在 $[-3, 3]$ 之间。也就是说，从概率上讲，无论原始分数的波动有多离谱，经过标准化后，95% 以上的优势值都会自然而然地落在 -2 到 +2 的区间内。
在 GRPO 实际训练中，为了节省显存，我们每次给同一个 Prompt 采样的回答数量 $G$ 通常很小（比如 $G=4$ 或 $G=8$）。在样本量极小的情况下，数学上有一个死定理叫做萨缪尔森不等式。它规定了：对于任意 $G$ 个样本，标准化后的最大绝对值，永远不可能超过 $\sqrt{G - 1}$。

## 1. 一句话

**标准 RL（GRPO/REINFORCE）只是"对正确轨迹做最大似然(MLE)"的一阶近似。**
MaxRL 指出：在"采样 + 二元对错反馈"的设定下，RL 的梯度其实是在对一个隐式似然做优化，
但 GRPO 用 `1/std` 归一优势，**反而更偏好简单题**，导致**分布锐化**（模型把概率质量
collapse 到少数高概率路径上，丢多样性）。MaxRL 只把优势的**分母从 std 换成"组均值 r̂"**
（一行代码），就让权重**单调地偏向难题**，逼近真正的 MLE → Pass@1 持平、**Pass@K 大涨**、
test-time scaling 效率比 GRPO 高 **7.9~19.2 倍**。

---

## 2. 为什么"RL 只是 MLE 的一阶近似"

把每个 prompt x 的"答对概率"记作 p_θ(x)（= 策略采样能产生正确答案的概率）。
我们真正想最大化的，是**对数似然** `log p_θ(x)`（让模型尽量答对）。它的梯度是：

```
∇ log p_θ(x) = (1 / p_θ(x)) · ∇ p_θ(x)         # 注意这个 1/p 权重
```

而标准 policy gradient / REINFORCE 实际优化的是 `p_θ(x)` 本身（或它的某种归一），梯度是：

```
∇ p_θ(x)                                        # 没有 1/p
```

两者只差一个 **`1/p_θ(x)` 的权重**。所以：

- **RL ≈ 最大化 p（一阶）**；**MLE = 最大化 log p**，多了 `1/p` 这个权重。
- `1/p` 的意义：**越难的题（p 越小）权重越大**。RL 没这个权重 → 难题学不动、容易被简单题带跑。

论文把它做成一个**按算力/采样预算 T 索引的目标族** `J^(T)`：
- **T=1** → 退化成标准 REINFORCE（一阶近似）；
- **T→∞** → 收敛到**精确 MLE**；
- 中间 T 用一个**无偏 policy-gradient 估计器**插值。

而**最实用的在线(on-policy)版本，就是把"1/p 权重"塞进优势计算里**——这就是那"一行改动"。

---

## 3. 那"一行改动"到底改什么

三种估计器对比（S_i = ∇_θ log π_θ(z_i|x) 是单条样本的 score function，r_i∈{0,1}）：

| 方法 | 梯度估计 | 优势的分母 |
|---|---|---|
| REINFORCE | `(1/N) Σ r_i S_i` | 除以**样本总数 N** |
| GRPO | `Σ (r_i-μ)/σ · S_i` | 除以**标准差 std** |
| **MaxRL** | `(1/K) Σ r_i S_i`，K=答对数 | 除以**组均值 r̂**（K=N·r̂） |

写成 verl 里"每条样本的优势 A_i"：

```
GRPO :  A_i = (r_i - r̂) / (std + eps)            # r̂=组均值, std=组标准差
MaxRL:  A_i = (r_i - r̂) / (r̂  + eps)             # ← 只把分母 std 换成组均值 r̂
```

**只把分母从 `std` 换成 `r̂`（组均值），就是全部改动。**

> 论文写成 `(r_i - r̂)/(N·r̂)`，那个 `1/N` 是"对 mini-batch 求和再归一"的系数；而 verl 的
> policy loss 本来就对样本取**均值**(= 自带 `1/N`)，所以代码里**不显式乘 N**——与 GRPO 优势
> `(r-μ)/σ` 不含 N 的写法一致。好处：MaxRL 与 GRPO 的优势量级在 p≈0.5 时都 ≈1，**同一 lr 即可
> 公平对照**。

### 权重函数对比（为什么 GRPO 偏好简单题、MaxRL 偏好难题）

二元奖励下 r̂=p（通过率），std=√(p(1-p))。看"答对样本"拿到的优势：

| | 答对样本的优势 ∝ | p→0（难题） | p→1（简单题） |
|---|---|---|---|
| GRPO | √((1-p)/p) | →∞（但只 ~1/√p） | →0 |
| **MaxRL** | (1-p)/p ∝ **1/p** | **→∞（更猛，~1/p）** | →0 |

- GRPO 的 `1/√(p(1-p))` 在 p→1 附近**反而回升**，等于给"几乎全对的简单题"也加权 → 鼓励
  模型把已经会的题刷得更尖 → **分布锐化、丢多样性**（Pass@K 变差）。
- MaxRL 的 `1/p` **单调**：越难越重、越简单越轻，正好是 MLE 想要的 `1/p` 权重 →
  学习信号集中在**低通过率的难题**上 → Pass@K、test-time scaling 大幅改善。

## 4. 在 verl 里怎么实现的（本仓库 `_my` 文件）

按你的规矩，只改了带 `_my` 后缀的副本，方便和原版 diff：

### (1) `verl/trainer/ppo/core_algos_my.py`
- 枚举加一项：`MAXRL = "maxrl"`。
- 新增 `compute_maxrl_outcome_advantage(...)`，注册名 `"maxrl"`。它和 `compute_grpo_outcome_advantage`
  **几乎逐行一样**，只有分母从 `id2std` 换成组均值 `id2mean`：
  ```python
  mean = id2mean[index[i]]               # r̂（二元时=通过率 p）
  if mean <= epsilon:                     # 组全错（r̂≈0）→ 无似然信号
      scores[i] = 0.0
  else:
      scores[i] = (scores[i] - mean) / (mean + epsilon)   # A_i = (r_i - r̂)/r̂
  ```
  并复用了 GRPO 那套 **genrm 软丢弃 `valid_mask`**（判分超时/失败的样本不进组统计、优势置 0）。

### (2) `verl/trainer/ppo/ray_trainer_my.py`
- `compute_advantage(...)` 里加了 `elif adv_estimator == AdvantageEstimator.MAXRL:` 分支，
  和 GRPO 分支一样传 `genrm_failed` 软丢弃，再调 `core_algos.compute_maxrl_outcome_advantage`。

> 这两个文件是"补丁暂存区"：要生效，按你现在 genrm 优势补丁一样的方式，把 `_my` 的改动同步/覆盖到
> 集群上真正被 `python -m verl.trainer.main_ppo` 加载的 `core_algos.py` / `ray_trainer.py`。

### (3) 训练脚本怎么开
把 `my-run_qwen3_5-27b-megatron-dapo.sh` 里：
```bash
adv_estimator=grpo      →      adv_estimator=maxrl
```
（`algorithm.adv_estimator` 是普通字符串，无枚举校验，直接填 `maxrl` 即可。）
其余 DAPO 配置（filter_groups、clip_ratio_c、token-mean…）都不用动——MaxRL 在周边逻辑里和
GRPO 完全等价（无 critic、无 REMAX baseline）。

## 5. 用的时候要注意（坑）

1. **奖励最好非负、理想是二元 0/1**。MaxRL 的 `1/r̂` 来自二元 MLE 推导。你的 genrm 给 0/0.5/1
   还行；但叠加 **overlong 惩罚**后单条奖励可能为负、组均值 r̂ 可能 ≈0 或为负 → `N·r̂` 失去
   "似然权重"意义。代码里对 `r̂≤0` 的组直接把优势置 0（这些组本就该被 `filter_groups` 滤掉）。
   想严格对齐论文，可考虑判分时只用纯 0/1 信号、把 overlong 惩罚挪到别处。
2. **难题优势会很大**：p 很小时 `1/p` 爆大，单条优势可能很大 → 靠 `clip_ratio_c`（你已设 10）
   和 `filter_groups` 兜住；若训练不稳，可在外层对 advantage 做一次裁剪。
3. **N 的取值**：代码里 N = 组内**有效**样本数（软丢弃后剩下的），与 r̂ 的统计口径一致。
4. **先小规模对照**：同数据、同超参，只切 `grpo` ↔ `maxrl`，比 **Pass@1 / Pass@K /
   多样性 / 训练稳定性**。论文卖点是 Pass@1 持平而 **Pass@K 与 test-time scaling 大涨**——
   所以评测一定要看 **Pass@K（k 取大，如 16/64）**，只看 Pass@1 看不出差别。

---

## 6. 预期效果（论文结论）

- **Pareto 优于 GRPO**：所有 benchmark 上 Pass@1 持平或更好，Pass@K 明显更高。
- **test-time scaling 效率 7.9~19.2×**：同样的 Pass@K 目标，MaxRL 训出的模型需要的采样预算
  小一个量级。
- 机理：似然式 `1/p` 重加权把信号压到难题上，避免 GRPO 的分布锐化、保住生成多样性。

---

## 7. 一句话回顾

> 想清楚一件事：**RL 在最大化"答对概率 p"，MLE 在最大化"log p"，差一个 `1/p` 权重。**
> MaxRL 把这个 `1/p` 通过"优势除以组均值 r̂ 而不是 std"补回来——**一行分母的改动**，
> 难题权重单调变大，逼近最大似然，换来更好的 Pass@K 和 test-time scaling。
